# 乐乐亲子乐园智能服务系统 — Dify 平台版

---

## 1. 项目概述

> **说明**：本项目为**学习练习用途**，所有乐园名称、项目、设施均为**完全虚拟**，不存在对应的实体景点。项目旨在通过构建一个完整的亲子主题乐园智能服务系统，练习 Dify 平台应用、Python 全栈开发、前后端分离架构等技术。

### 1.1 项目背景

本项目基于原有 Coze 平台的「亲子主题乐园智能服务系统」进行**全面升级与平台迁移**，将系统从 Coze 平台迁移至 **Dify** 开源 LLM 应用开发平台，同时扩大项目规模，构建一套完整的**全栈智能乐园服务系统**。

升级后的系统具备以下特征：

- **AI 平台**：从 Coze 迁移至 Dify，利用 Dify 的 Workflow、Agent、知识库、工具调用等能力
- **后端系统**：基于 Python（FastAPI）构建完整的后端服务，提供 RESTful API + WebSocket 实时通信
- **前端系统**：Web 端（Vue 3）+ 微信小程序（uni-app），覆盖 PC、移动端、微信生态
- **功能扩展**：新增在线购票、会员系统、智能排队、社区互动等功能模块

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **平台迁移** | 将 Coze 多 Agent 架构完整迁移至 Dify 平台，保留并增强原有智能服务能力 |
| **全栈构建** | 构建 Python 后端 + Vue 3 Web 前端 + 微信小程序的完整前后端系统 |
| **功能扩展** | 新增购票、会员、排队、社区等功能，练习真实业务场景的系统设计 |
| **技术学习** | 练习 Dify 平台配置、FastAPI 开发、Vue 3 前端、WebSocket 实时通信等技术 |

### 1.3 系统定位

本系统为**虚拟亲子主题乐园**的学习项目，模拟为游客（家长+儿童）和园区运营方提供一站式智能服务平台：

- **游客端**：AI 对话助手、在线购票、路线规划、实时排队、社区分享
- **运营端**：数据看板、活动管理、用户分析

## 4. 技术栈总览

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| **AI 平台** | Dify（开源版/云版） | Workflow + Agent + 知识库 + 工具调用 |
| **后端框架** | FastAPI + Uvicorn | 高性能异步 Python Web 框架 |
| **数据库** | MySQL 8.0 | 关系型主数据库 |
| **缓存** | Redis 7.x | 会话缓存、排队数据、限流 |
| **向量数据库** | Milvus / Weaviate | Dify 知识库向量检索 |
| **ORM** | SQLAlchemy 2.0 + Alembic | 数据库 ORM + 迁移管理 |
| **Web 前端** | Vue 3 + Vite + Element Plus | PC 端 + 移动端响应式 |
| **小程序** | uni-app (Vue 3) | 微信小程序，跨平台复用 |
| **实时通信** | WebSocket (FastAPI WebSocket) | AI 对话流式输出、排队推送 |
| **文件存储** | MinIO / 阿里云 OSS | 图片、视频、文档存储 |
| **部署** | Docker + Docker Compose | 容器化部署 |
| **反向代理** | Nginx | SSL 终止、负载均衡、静态资源 |
| **监控** | Prometheus + Grafana | 系统监控与告警 |

## 5. Coze → Dify 迁移概览

| Coze 组件 | Dify 对应组件 | 迁移策略 |
|-----------|--------------|----------|
| Coze Bot（多 Agent 模式） | Dify Chatflow / Agent | 重建 Agent 架构，适配 Dify 路由机制 |
| Coze Workflow（6个） | Dify Workflow | 节点逐一映射，调整 API 调用方式 |
| Coze Knowledge Base（7个） | Dify Knowledge Base | 文档导入 + 向量化配置 |
| Coze Database（5张表） | 外部 MySQL（后端管理） | 数据迁移到独立 MySQL，Dify 通过 API 工具调用 |
| Coze Plugin | Dify Tool（自定义工具） | 对接后端 API 作为 Dify 自定义工具 |
| Coze API | Dify API | 后端通过 Dify API 触发对话/工作流 |

> 详细迁移方案见 [02-Dify平台迁移方案.md](./02-Dify平台迁移方案.md)
