# 系统架构设计

> **职责**：定义系统整体架构、技术选型依据、部署架构、网络拓扑
> **说明**：本项目为学习练习用途，所有乐园内容均为虚拟，不对应实体景点。
> **关联**：[README.md](./README.md)

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **前后端分离** | 前端（Web/小程序）与后端（Python）通过 API 通信，独立开发部署 |
| **AI 平台解耦** | Dify 作为独立 AI 服务层，后端通过 API 调用，可替换为其他 LLM 平台 |
| **微服务化** | 后端按业务领域拆分为独立服务模块，松耦合高内聚 |
| **数据驱动** | 所有业务数据沉淀到数据库，支撑分析和决策 |
| **安全优先** | 用户数据加密、API 鉴权、儿童信息保护 |
| **可扩展** | 架构支持水平扩展，模块可独立升级 |

## 2. 技术选型详细说明

### 2.1 AI 平台：Dify

**选择理由**：

| 对比维度 | Dify | Coze |
|----------|------|------|
| 开源 | ✅ 开源可自部署 | ❌ 闭源 SaaS |
| 数据隐私 | ✅ 数据可完全私有化 | ❌ 数据在字节跳动服务器 |
| LLM 选择 | ✅ 支持 OpenAI/Claude/本地模型 | ❌ 仅限 Coze 内置模型 |
| 自定义工具 | ✅ API 工具 + 代码工具 | ⚠️ 插件市场为主 |
| Workflow | ✅ 可视化 Workflow 编辑器 | ✅ 类似 |
| 知识库 | ✅ 多种向量化引擎可选 | ⚠️ 固定引擎 |
| API 集成 | ✅ 完善的 RESTful API | ✅ 有 API |
| 部署方式 | ✅ Docker 自部署 / 云版 | ❌ 仅 SaaS |
| 成本 | ✅ 社区版免费 | ⚠️ 按量付费 |

**部署方式推荐**：
- 开发/测试阶段：Dify 云版（快速启动）
- 生产环境：Dify 开源版 Docker 自部署（数据隐私 + 成本可控）

### 2.2 后端：FastAPI

**选择理由**：
- Python 生态丰富，与 AI/ML 工具链无缝衔接
- 原生异步支持（async/await），高并发性能优异
- 自动生成 OpenAPI 文档，前后端协作高效
- 类型安全（Pydantic），减少运行时错误

**版本**：Python 3.11+ / FastAPI 0.110+

### 2.3 前端：Vue 3 + uni-app

**选择理由**：
- Vue 3 生态成熟，Composition API 开发体验好
- uni-app 支持一套代码编译到微信小程序 + H5，减少重复开发
- Element Plus 组件库丰富，适合管理后台
- Vite 构建速度快

### 2.4 数据库：MySQL + Redis

**MySQL 8.0**：主业务数据库，支持 JSON 字段、窗口函数、CTE
**Redis 7.x**：会话缓存、实时排队数据、限流计数器、消息队列

## 6. 安全架构

### 6.1 认证与授权

| 机制 | 说明 |
|------|------|
| **JWT Token** | 用户登录后颁发 access_token (2h) + refresh_token (7d) |
| **微信 OAuth** | 小程序端通过微信登录获取 openid，后端生成 JWT |
| **API Key** | 后端 → Dify 使用 API Key 鉴权 |
| **RBAC** | 运营后台基于角色的权限控制（管理员/运营/客服） |

### 6.2 数据安全

| 措施 | 说明 |
|------|------|
| **传输加密** | 全站 HTTPS，WebSocket 使用 WSS |
| **数据加密** | 用户密码 bcrypt 加密，敏感字段 AES 加密 |
| **儿童信息保护** | 儿童信息脱敏存储，不用于商业用途 |
| **SQL 注入防护** | SQLAlchemy 参数化查询 |
| **XSS 防护** | 前端输入过滤 + CSP 头 |
| **CSRF 防护** | SameSite Cookie + CSRF Token |
| **限流** | Redis 实现 API 限流（每用户 60次/分钟） |
| **日志审计** | 关键操作记录审计日志 |

## 7. 性能设计

| 策略 | 说明 |
|------|------|
| **缓存** | Redis 缓存热点数据（项目信息、排队数据、票价），TTL 5-30 分钟 |
| **数据库优化** | 合理索引、读写分离、分页查询 |
| **CDN** | 静态资源走 CDN 加速 |
| **WebSocket** | AI 对话、排队推送使用 WebSocket 长连接 |
| **异步任务** | 短信发送、数据分析等使用异步队列 |
| **API 合并** | 前端使用 BFF 模式合并多个接口请求 |
