## 2. 分段策略详情

### 2.1 kb-projects（乐园项目大全）

**分段模式**：自定义分段（按 `##` 二级标题）

**每个 chunk 包含**：
- 项目名称 + 编号
- 区域、适合年龄、身高要求
- 体力强度、室内/室外、婴儿车友好
- 游玩时长、排队支持、快速通道
- 项目简介
- 游玩提示

**chunk 示例**：

```
## P001 · 旋转木马
区域：梦幻城堡区（中心位置）
适合年龄：全年龄（含0岁婴儿，家长陪同即可）
身高要求：无限制
体力强度：低
室内/室外：室外（有遮阳顶棚）
婴儿车友好：是（有婴儿车停放区）
预计游玩时长：约10分钟
排队&快速通道：支持排队，不支持快速通道
当前状态：开放中

项目简介：经典童话风格旋转木马，双层设计...

游玩提示：
- 适合低龄宝宝首次体验，家长可陪同乘坐
- 每匹马限坐1位儿童+1位家长陪同
...
```

### 2.2 kb-age-guide（各年龄段游玩指南）

**分段模式**：按年龄段 + 兴趣分类

**chunk 划分**：
1. 总览：年龄分段与游玩策略
2. 0-3岁：发育特征 + 推荐项目
3. 0-3岁：一日游推荐路线
4. 0-3岁：安全与注意事项
5. 3-6岁：发育特征 + 推荐项目（动物/表演/冒险）
6. 3-6岁：一日游推荐路线
7. 6-12岁：发育特征 + 推荐项目（刺激/科普/表演）
8. 6-12岁：一日游推荐路线
9. 多年龄段家庭游玩建议
10. 共同项目兼容性表

### 2.3 kb-faq（FAQ 常见问题解答）

**分段模式**：QA 模式（自动提取问答对）

每个 Q-A 对作为一个 chunk：

```
Q: 乐园营业时间是什么时候？
A: 平时（周一至周五）09:00-18:00，周末及法定节假日 09:00-20:00。停车场开放 07:30-20:00（周末延长至 22:00）。
```

## 3. 向量化配置

### 3.1 Embedding 模型选择

| 场景 | 推荐模型 | 说明 |
|------|----------|------|
| **Dify 云版** | text-embedding-3-small (OpenAI) | 简单易用，中文支持好 |
| **自部署（推荐）** | bge-large-zh-v1.5 (BAAI) | 中文效果最佳，本地运行 |
| **备选** | m3e-base | 开源中文模型，资源消耗小 |

### 3.2 检索配置

```yaml
检索模式: 混合检索（向量检索 + 全文检索）
权重分配:
  向量检索: 0.7
  全文检索: 0.3
TopK: 5
Score 阈值: 0.5
Rerank: 开启
Rerank 模型: bge-reranker-v2-m3
```

### 3.3 分段参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 分段标识符 | `##` | 按二级标题分段 |
| 最大分段长度 | 1000 tokens | 避免 chunk 过大 |
| 分段重叠 | 100 tokens | 保持上下文连贯 |
| 清洗规则 | 去除多余空行、统一标点 | 提高检索质量 |

### 4.3 知识库同步脚本

```python
# scripts/sync_knowledge.py
"""
将本地知识文档同步到 Dify 知识库
运行方式: python -m scripts.sync_knowledge
前置条件: Dify 平台已部署，API Key 已配置
"""
import asyncio
import os
import httpx
from pathlib import Path

DIFY_API_BASE = os.getenv("DIFY_API_BASE", "http://localhost/v1")
DIFY_DATASET_API_KEY = os.getenv("DIFY_DATASET_API_KEY", "dataset-xxxxx")

KNOWLEDGE_DIR = Path(__file__).parent.parent / "知识库"

KNOWLEDGE_FILES = [
    {"name": "kb-projects", "file": "01-乐园项目大全.md", "segment_mode": "custom", "delimiter": "##"},
    {"name": "kb-age-guide", "file": "02-各年龄段游玩指南.md", "segment_mode": "custom", "delimiter": "##"},
    {"name": "kb-safety", "file": "03-安全游玩规则.md", "segment_mode": "custom", "delimiter": "##"},
    {"name": "kb-facilities", "file": "04-园区便利设施指南.md", "segment_mode": "custom", "delimiter": "##"},
    {"name": "kb-faq", "file": "05-FAQ常见问题解答.md", "segment_mode": "qa"},
    {"name": "kb-events", "file": "06-演出与活动指南.md", "segment_mode": "custom", "delimiter": "##"},
    {"name": "kb-weather", "file": "07-天气应对指南.md", "segment_mode": "custom", "delimiter": "##"},
]

HEADERS = {
    "Authorization": f"Bearer {DIFY_DATASET_API_KEY}",
    "Content-Type": "application/json",
}

async def create_dataset(client: httpx.AsyncClient, name: str) -> str:
    """创建知识库（如不存在），返回 dataset_id"""
    # 查询已有知识库
    resp = await client.get(f"{DIFY_API_BASE}/datasets", params={"page": 1, "limit": 100}, headers=HEADERS)
    datasets = resp.json().get("data", [])
    for ds in datasets:
        if ds["name"] == name:
            print(f"  知识库已存在: {name} (id={ds['id']})")
            return ds["id"]

    # 创建新知识库
    resp = await client.post(f"{DIFY_API_BASE}/datasets", json={"name": name}, headers=HEADERS)
    dataset_id = resp.json()["id"]
    print(f"  创建知识库: {name} (id={dataset_id})")
    return dataset_id


async def upload_document(client: httpx.AsyncClient, dataset_id: str, file_path: Path, kb_config: dict) -> dict:
    """上传文档到知识库"""
    with open(file_path, "rb") as f:
        files = {"file": (file_path.name, f, "text/markdown")}
        data = {
            "name": file_path.stem,
            "indexing_technique": "high_quality",
            "process_rule": "{}",  # 使用默认规则
        }
        resp = await client.post(
            f"{DIFY_API_BASE}/datasets/{dataset_id}/document/create-by-text",
            json={
                "name": file_path.stem,
                "text": f.read().decode("utf-8"),
                "indexing_technique": "high_quality",
                "process_rule": {
                    "mode": "custom" if kb_config.get("segment_mode") == "custom" else "automatic",
                    "rules": {
                        "pre_processing_rules": [
                            {"id": "remove_extra_spaces", "enabled": True},
                            {"id": "remove_urls_emails", "enabled": False},
                        ],
                        "segmentation": {
                            "separator": kb_config.get("delimiter", "\n\n"),
                            "max_tokens": 1000,
                        },
                    } if kb_config.get("segment_mode") == "custom" else None,
                },
            },
            headers=HEADERS,
        )
    return resp.json()


async def sync_all():
    """同步所有知识文档到 Dify"""
    print("=" * 50)
    print("  知识库同步工具")
    print("=" * 50)

    async with httpx.AsyncClient(timeout=60.0) as client:
        for kb in KNOWLEDGE_FILES:
            file_path = KNOWLEDGE_DIR / kb["file"]
            if not file_path.exists():
                print(f"  ⚠️ 文件不存在: {file_path}")
                continue

            print(f"\n📚 同步 {kb['name']}...")
            try:
                dataset_id = await create_dataset(client, kb["name"])
                result = await upload_document(client, dataset_id, file_path, kb)
                print(f"  ✅ 上传成功: {kb['file']} → {kb['name']}")
            except Exception as e:
                print(f"  ❌ 同步失败: {e}")

    print(f"\n{'=' * 50}")
    print("  同步完成！")
    print(f"{'=' * 50}")

if __name__ == "__main__":
    asyncio.run(sync_all())
```

## 5. 知识库更新策略

| 场景 | 更新方式 | 频率 |
|------|----------|------|
| 项目信息变更 | 运营后台编辑 → API 更新 Dify 知识库 | 实时 |
| 活动排期更新 | 定时任务自动同步 | 每日凌晨 |
| 新增 FAQ | 运营后台添加 → API 更新 | 实时 |
| 安全规则更新 | 运营后台编辑 → API 更新 | 按需 |
| 票价调整 | 运营后台修改 → 自动同步知识库 | 实时 |

## 6. 数据备份策略

| 数据 | 备份方式 | 频率 | 保留时间 |
|------|----------|------|---------|
| MySQL 全量 | mysqldump | 每日凌晨 | 30 天 |
| MySQL binlog | 实时同步 | 持续 | 7 天 |
| Redis | RDB + AOF | 每小时 | 7 天 |
| Dify 知识库 | 文档源文件 Git 管理 | 每次变更 | 永久 |
| 用户上传文件 | OSS 跨区域复制 | 实时 | 永久 |
