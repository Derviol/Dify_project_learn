# 数据库设计

> **职责**：定义完整的数据库表结构、ER 关系、索引策略
> **说明**：本项目为学习练习用途，所有数据均为虚拟。
> **关联**：[03-后端技术方案.md](./03-后端技术方案.md)

---

## 1. 总览

共设计 **16 张核心业务表**，覆盖用户、票务、游玩、排队、会员、社区、运营全部业务领域。

| 领域 | 表数量 | 表名 |
|------|:------:|------|
| 用户 | 2 | users, user_profiles |
| 票务 | 3 | ticket_types, ticket_orders, ticket_items |
| 项目 | 2 | projects, facilities |
| 排期 | 1 | schedules |
| 排队 | 1 | queue_records |
| 会员 | 2 | members, point_records |
| 社区 | 2 | community_posts, post_reviews |
| 安全 | 1 | safety_rules |
| 运营 | 2 | notifications, operation_logs |

## 3. 表结构详细设计

### 3.1 users — 用户表

```sql
CREATE TABLE users (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    phone           VARCHAR(20) UNIQUE,              -- 手机号（可选登录方式）
    email           VARCHAR(100) UNIQUE,             -- 邮箱（可选）
    password_hash   VARCHAR(255),                    -- 密码哈希（手机/邮箱登录时）
    nickname         VARCHAR(50) NOT NULL,            -- 昵称
    avatar_url      VARCHAR(500),                    -- 头像 URL
    wechat_openid   VARCHAR(100) UNIQUE,             -- 微信 OpenID
    wechat_unionid  VARCHAR(100),                    -- 微信 UnionID
    role            ENUM('user','admin','staff') DEFAULT 'user',
    status          ENUM('active','banned','deleted') DEFAULT 'active',
    last_login_at   DATETIME,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_phone (phone),
    INDEX idx_wechat_openid (wechat_openid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.2 user_profiles — 用户画像表

```sql
CREATE TABLE user_profiles (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT UNIQUE NOT NULL,
    child_age       VARCHAR(20),                     -- 孩子年龄（如 "3岁"）
    child_gender    ENUM('male','female','unknown') DEFAULT 'unknown',
    interests       JSON,                            -- 兴趣偏好 ["动物","表演","冒险"]
    visit_count     INT DEFAULT 0,                   -- 游玩次数
    special_needs   TEXT,                            -- 特殊需求（过敏/行动不便）
    preferred_zones JSON,                            -- 偏好区域
    last_visit_date DATE,                            -- 上次游玩日期
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.3 projects — 项目/设施表

```sql
CREATE TABLE projects (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    project_code    VARCHAR(20) UNIQUE NOT NULL,      -- 项目编号（P001-P015）
    name            VARCHAR(100) NOT NULL,            -- 项目名称
    zone            VARCHAR(50) NOT NULL,             -- 所在区域
    category        ENUM('ride','venue','show','water') NOT NULL,  -- 类型
    description     TEXT,                             -- 项目简介
    suitable_age    VARCHAR(50),                      -- 适合年龄
    min_height      DECIMAL(3,2),                     -- 最低身高（米）
    max_height      DECIMAL(3,2),                     -- 最高身高（米）
    intensity       ENUM('low','medium','high') DEFAULT 'low',  -- 体力强度
    is_indoor       BOOLEAN DEFAULT FALSE,            -- 是否室内
    has_ac          BOOLEAN DEFAULT FALSE,            -- 是否有空调
    stroller_friendly BOOLEAN DEFAULT TRUE,           -- 婴儿车友好
    duration_minutes INT,                             -- 预计游玩时长
    capacity        INT,                              -- 容纳人数
    fast_pass       BOOLEAN DEFAULT FALSE,            -- 是否支持快速通道
    fast_pass_price DECIMAL(10,2),                    -- 快速通道价格
    queue_enabled   BOOLEAN DEFAULT TRUE,             -- 是否支持排队
    cover_image     VARCHAR(500),                     -- 封面图片
    location_desc   VARCHAR(200),                     -- 位置描述（如"梦幻城堡区中心位置"）
    tips            TEXT,                             -- 游玩提示
    safety_notes    TEXT,                             -- 安全注意事项
    status          ENUM('open','closed','maintenance') DEFAULT 'open',
    sort_order      INT DEFAULT 0,                    -- 排序权重
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_zone (zone),
    INDEX idx_category (category),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.4 facilities — 便利设施表

```sql
CREATE TABLE facilities (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    name            VARCHAR(100) NOT NULL,            -- 设施名称
    type            ENUM('restroom','nursing','restaurant','snack','locker',
                         'charging','stroller','wheelchair','medical',
                         'info','water','parking') NOT NULL,
    zone            VARCHAR(50),                      -- 所在区域
    location_desc   VARCHAR(200),                     -- 位置描述（如"旋转木马旁，步行约3分钟"）
    open_time       TIME,                             -- 开放时间
    close_time      TIME,
    features        JSON,                             -- 设施特点 ["哺乳隔间","换尿布台","热水"]
    price_info      VARCHAR(200),                     -- 价格信息
    contact_phone   VARCHAR(20),                      -- 联系电话
    status          ENUM('open','closed','maintenance') DEFAULT 'open',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_type (type),
    INDEX idx_zone (zone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.5 schedules — 活动排期表

```sql
CREATE TABLE schedules (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    project_id      BIGINT NOT NULL,                  -- 关联项目
    event_name      VARCHAR(100) NOT NULL,            -- 活动名称
    event_type      ENUM('show','parade','workshop','special') NOT NULL,
    event_date      DATE NOT NULL,                    -- 活动日期
    start_time      TIME NOT NULL,                    -- 开始时间
    end_time        TIME NOT NULL,                    -- 结束时间
    location        VARCHAR(100),                     -- 活动地点
    capacity        INT,                              -- 容纳人数
    enrolled_count  INT DEFAULT 0,                    -- 已报名人数
    theme           VARCHAR(100),                     -- 场次主题
    description     TEXT,                             -- 活动描述
    notes           TEXT,                             -- 备注
    status          ENUM('normal','cancelled','full') DEFAULT 'normal',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_date (event_date),
    INDEX idx_project (project_id),
    FOREIGN KEY (project_id) REFERENCES projects(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.6 ticket_types — 票种表

```sql
CREATE TABLE ticket_types (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    name            VARCHAR(50) NOT NULL,             -- 票种名称
    code            VARCHAR(20) UNIQUE NOT NULL,      -- 票种编码
    category        ENUM('adult','child','family','senior','annual') NOT NULL,
    price           DECIMAL(10,2) NOT NULL,           -- 原价
    discount_price  DECIMAL(10,2),                    -- 优惠价
    description     TEXT,                             -- 票种说明
    includes        JSON,                             -- 包含内容 ["成人×2","儿童×1"]
    valid_days      INT DEFAULT 1,                    -- 有效天数
    usage_limit     INT DEFAULT 1,                    -- 使用次数限制
    status          ENUM('active','inactive') DEFAULT 'active',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.7 ticket_orders — 票务订单表

```sql
CREATE TABLE ticket_orders (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no        VARCHAR(32) UNIQUE NOT NULL,      -- 订单号
    user_id         BIGINT NOT NULL,
    total_amount    DECIMAL(10,2) NOT NULL,           -- 订单总额
    pay_amount      DECIMAL(10,2) NOT NULL,           -- 实付金额
    pay_method      ENUM('wechat','alipay','cash') DEFAULT 'wechat',
    pay_status      ENUM('unpaid','paid','refunding','refunded') DEFAULT 'unpaid',
    visit_date      DATE NOT NULL,                    -- 游玩日期
    status          ENUM('pending','confirmed','used','cancelled','expired') DEFAULT 'pending',
    qr_code         VARCHAR(500),                     -- 电子票二维码内容
    used_at         DATETIME,                         -- 使用时间
    cancelled_at    DATETIME,
    refund_amount   DECIMAL(10,2),
    refund_reason   VARCHAR(200),
    remark          VARCHAR(500),
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_user (user_id),
    INDEX idx_order_no (order_no),
    INDEX idx_visit_date (visit_date),
    INDEX idx_status (status),
    FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.8 ticket_items — 订单明细表

```sql
CREATE TABLE ticket_items (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_id        BIGINT NOT NULL,
    ticket_type_id  BIGINT NOT NULL,
    quantity        INT NOT NULL DEFAULT 1,
    unit_price      DECIMAL(10,2) NOT NULL,
    subtotal        DECIMAL(10,2) NOT NULL,
    visitor_name    VARCHAR(50),                      -- 使用人姓名
    visitor_id_no   VARCHAR(50),                      -- 使用人证件号（加密存储）
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (order_id) REFERENCES ticket_orders(id) ON DELETE CASCADE,
    FOREIGN KEY (ticket_type_id) REFERENCES ticket_types(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.9 queue_records — 排队记录表

```sql
CREATE TABLE queue_records (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL,
    project_id      BIGINT NOT NULL,
    queue_number    INT NOT NULL,                     -- 排队号码
    queue_date      DATE NOT NULL,                    -- 排队日期
    status          ENUM('waiting','called','serving','completed','cancelled','expired') DEFAULT 'waiting',
    taken_at        DATETIME DEFAULT CURRENT_TIMESTAMP,
    called_at       DATETIME,
    completed_at    DATETIME,
    cancelled_at    DATETIME,

    INDEX idx_user_date (user_id, queue_date),
    INDEX idx_project_date (project_id, queue_date),
    INDEX idx_status (status),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (project_id) REFERENCES projects(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.10 members — 会员表

```sql
CREATE TABLE members (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT UNIQUE NOT NULL,
    member_no       VARCHAR(20) UNIQUE NOT NULL,      -- 会员编号
    level           INT DEFAULT 1,                    -- 会员等级（1-5）
    points          INT DEFAULT 0,                    -- 当前积分
    total_points    INT DEFAULT 0,                    -- 累计积分
    balance         DECIMAL(10,2) DEFAULT 0.00,       -- 储值余额
    card_type       ENUM('none','silver','gold','diamond') DEFAULT 'none',
    card_expire     DATE,                             -- 年卡到期日
    join_date       DATE DEFAULT (CURRENT_DATE),
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.11 point_records — 积分记录表

```sql
CREATE TABLE point_records (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL,
    member_id       BIGINT NOT NULL,
    points          INT NOT NULL,                     -- 积分变动（正=获得，负=消耗）
    balance_after   INT NOT NULL,                     -- 变动后余额
    type            ENUM('earn_purchase','earn_checkin','earn_review',
                         'earn_referral','spend_gift','spend_discount') NOT NULL,
    description     VARCHAR(200),
    related_order   VARCHAR(32),                      -- 关联订单号
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,

    INDEX idx_user (user_id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (member_id) REFERENCES members(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3.12 community_posts — 社区帖子表

```sql
CREATE TABLE community_posts (
```

```sql
CREATE TABLE post_reviews (
```

## 4. 索引策略

| 表 | 核心索引 | 说明 |
|----|---------|------|
| users | phone, wechat_openid | 登录查询 |
| projects | zone, category, status | 项目筛选 |
| ticket_orders | user_id + status, order_no, visit_date | 订单查询 |
| queue_records | project_id + queue_date + status | 排队查询（热数据） |
| schedules | event_date | 排期查询 |
| community_posts | user_id, post_type, created_at | 社区列表 |

> 排队数据高频读写，核心数据缓存在 Redis，MySQL 仅做持久化。

### 5.1 Alembic 迁移配置

```bash
# 初始化 Alembic
alembic init alembic

# 生成初始迁移
alembic revision --autogenerate -m "initial_schema"

# 执行迁移
alembic upgrade head
```

**alembic.ini 关键配置**：

```ini
[alembic]
script_location = alembic
sqlalchemy.url = mysql+pymysql://root:password@localhost:3306/lele_park
```

**alembic/env.py 关键配置**：

```python
from app.models.base import Base
target_metadata = Base.metadata
```
